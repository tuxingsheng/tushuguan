# 常见问题

## 页面布局没铺满

**现象：** 内容区没延伸到页面底部，或 `n-layout--static-positioned` 高度为 0。

**原因：** Naive UI `n-layout` 默认 `position: static`，不遵循标准 flex 布局。

**解决：** 已在 `global.css` 中修复。如果复发，检查：
1. `html, body, #app` 是否设置了 `height: 100%`
2. 内层 layout 是否用了 `height: calc(100vh - 52px)` 而非 `flex: 1`

## 仪表盘显示全部为 0

**可能原因：**

1. **Supabase 未建表** → 查看 Dashboard 是否显示错误信息。若有错误卡片，点击"重新加载"确认。
2. **确实没有数据** → 会显示引导卡片，按步骤添加分类 → 图书 → 读者即可。
3. **网络问题** → 打开浏览器开发者工具 → Network 面板，查看 Supabase API 请求是否返回错误。

## 仪表盘加载失败

Dashboard 现在会显示具体错误信息（如 "relation 'books' does not exist"）和"重新加载"按钮，不再静默失败。

## 借书时提示"该书已无库存"

`borrow_book` RPC 在数据库层面检查 `available > 0`。如果前端列表显示有库存但借阅失败，是并发问题——另一个管理员刚借走了最后一本。刷新页面即可看到最新库存。

## 删除图书/读者后借阅记录消失

`borrowings` 表外键设置了 `ON DELETE CASCADE`，删除图书或读者会自动删除关联借阅记录。删除确认框中已提示此影响。

## IDE 提示 "All imports are unused"

`<script setup>` 中导入的 Naive UI 组件自动注册到模板，IDE 无法检测到模板中的使用。这是误报，可忽略。

## 列表页数据为空时一片空白

所有列表页（Books、Members、Borrowings、Categories）均已添加 `<n-empty>` 空状态提示。如果搜索后无结果，会显示"暂无数据"而非空白表格。

## 搜索输入 % 或 _ 导致意外结果

搜索关键词直接拼接到 Supabase `ilike` 查询中，`%` 和 `_` 会被解释为 SQL LIKE 通配符。例如搜索 "100%" 会匹配所有 "100" 开头的记录。目前未做转义处理。

## 编辑图书时库存改成小于借出数

`update_book_stock` RPC 会检查 `新库存 - 当前借出数 >= 0`，不满足时抛异常并提示"库存不能小于当前借出数量"。用户会看到具体错误消息。
