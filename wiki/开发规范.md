# 开发规范

## 命名

| 类别 | 规则 | 示例 |
|---|---|---|
| 组件文件 | PascalCase | `BookForm.vue`、`AppLayout.vue` |
| JS/Store 文件 | camelCase | `supabase.js`、`auth.js` |
| 目录 | 小写单数 | `stores`、`views`、`components` |
| 路由路径 | 小写 + 短横线 | `/books/add`、`/books/:id/edit` |
| 变量/函数 | camelCase | `loadBooks`、`handleSubmit`、`selectedCategory` |
| 事件处理函数 | `handleXxx` | `handleLogin`、`handleDelete`、`handleSearch` |
| 数据加载函数 | `loadXxx` / `fetchXxx` | `loadBooks`、`fetchBook` |

## 组件通信

- **父子**：props down, events up
- **跨组件**：Pinia Store
- **路由参数**：`useRoute().params.id`
- **消息通知**：`useMessage()`（须在 `<n-message-provider>` 内）

## Supabase 调用

- ✅ CRUD 操作走 Store 方法，View 不直接调 Supabase
- ✅ 数据一致性操作（借阅/归还/库存）必须用 RPC
- ✅ 删除操作必须 `n-popconfirm` 二次确认
- ✅ 错误在 Store 中 throw，由 View 层 `try/catch` + `useMessage()` 处理
- ❌ 不在 View 中直接 `supabase.from().insert()`

## CSS

- 全局样式在 `global.css` 的 CSS 变量中定义
- 组件内用 `<style scoped>`
- Naive UI 覆写用 `:deep()` 选择器
- 内联样式仅用于动态计算的数值（如 `calc(100vh - 52px)`）
- 避免全局 CSS 覆盖组件库内部类名（优先级不够）

## 错误处理

| 场景 | 方式 |
|---|---|
| CRUD 操作失败 | View 层 try/catch → `message.error(err.message)` |
| 数据加载失败（Dashboard） | 捕获错误 → 显示 `n-result` 错误页 + 重试按钮 |
| 表单加载失败（BookForm/MemberForm） | 提示错误 + 跳转到列表页 |
| 表单验证失败 | Naive UI `n-form` rules 自动拦截 |

## 移动端 / H5

- 使用 `window.innerWidth < 768` 判断移动端
- 响应式断点：768px（平板/手机）、480px（小屏手机）
- 表格需包裹 `overflow-x: auto` 支持横向滑动
- 按钮 `min-height: 40px` 满足触摸友好
- 弹窗在移动端 `max-width: 95vw`
- 测试时在 Chrome DevTools 切换设备模式验证
