# 路由设计

## 路由表

| 路径 | 组件 | 认证 | 说明 |
|---|---|---|---|
| `/login` | Login | 否 | 管理员登录 |
| `/` | Dashboard | 是 | AppLayout 为父组件 |
| `/books` | Books | 是 | 图书列表 |
| `/books/add` | BookForm | 是 | 新增图书 |
| `/books/:id/edit` | BookForm | 是 | 编辑图书 |
| `/members` | Members | 是 | 读者列表 |
| `/members/add` | MemberForm | 是 | 新增读者 |
| `/members/:id/edit` | MemberForm | 是 | 编辑读者 |
| `/borrowings` | Borrowings | 是 | 借阅记录 |
| `/borrowings/new` | BorrowForm | 是 | 新建借阅 |
| `/categories` | Categories | 是 | 分类管理 |

## 导航守卫

```js
router.beforeEach(async (to, from, next) => {
  const { data: { session } } = await supabase.auth.getSession()

  if (to.meta.requiresAuth !== false && !session) {
    next('/login')        // 未登录 → 强制跳转
  } else if (to.path === '/login' && session) {
    next('/')             // 已登录 → 不重复登录
  } else {
    next()
  }
})
```

**注意：** 守卫仅在导航时触发。若会话在页面停留期间过期，后续 API 调用返回 401，但用户不会自动跳转登录页，直到下一次页面跳转。项目未注册 `supabase.auth.onAuthStateChange`。

## 侧栏菜单激活规则

`AppLayout.vue` 中 `activeKey` 通过 `route.path` 前缀匹配：

| 路径前缀 | 菜单 Key |
|---|---|
| `/books` | `books` |
| `/members` | `members` |
| `/borrowings` | `borrowings` |
| `/categories` | `categories` |
| 其他 `/` | `dashboard` |

## 路由结构

```
/login          → Login.vue（独立，无布局）
/               → AppLayout.vue（父布局）
  ├─ /          → Dashboard.vue
  ├─ /books     → Books.vue
  ├─ /books/add → BookForm.vue
  ├─ /books/:id/edit → BookForm.vue
  ├─ /members   → Members.vue
  ├─ ...
```
